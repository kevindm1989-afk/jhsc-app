# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in this project, please report it
responsibly:

**Email:** security@[your-domain].ca

Please include:
- A description of the vulnerability
- Steps to reproduce
- The impact you've identified
- Any proof-of-concept code (please don't publish this publicly)

We commit to:
- Acknowledging your report within 48 hours
- Providing a timeline for investigation within 5 business days
- Keeping you informed of progress
- Crediting you in any disclosure (unless you prefer anonymity)

Please do **not**:
- Publicly disclose the vulnerability before we've had a chance to investigate
  and fix it
- Test the vulnerability against production systems with real user data
- Use the vulnerability to access data beyond what's necessary to demonstrate
  the issue

## PIPEDA breach reporting

Under PIPEDA, we're required to notify the Office of the Privacy Commissioner
of Canada and affected individuals of any breach of security safeguards
involving personal information where there's a real risk of significant harm.

If you believe your personal information may have been affected by a security
incident, contact: privacy@[your-domain].ca

We maintain a breach register per PIPEDA s.10.1 requirements.

## Scope

In scope for responsible disclosure:
- This codebase
- Production deployments at [your-domain].ca

Out of scope:
- Third-party services we depend on (please report to them directly)
- Social engineering of staff
- Physical security
- Denial-of-service attacks

## Safe harbor

We will not pursue legal action against researchers who:
- Make a good-faith effort to follow this policy
- Avoid privacy violations and data destruction
- Report vulnerabilities promptly
- Do not extort or threaten the project
